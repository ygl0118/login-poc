# 手動測試案例 — 登入流程

對應目標：`https://swag.live/?lang=zh-TW`。優先度 P0（核心）～ P2（邊界）。
帳密：由環境變數 `LOGIN_USERNAME` / `LOGIN_PASSWORD` 提供（不寫死於文件，見 README「憑證管理」）。

## TC-01（P0）正確帳密登入成功
**前置**：已開啟瀏覽器、未登入。
**步驟**：
1. 進站，於 landing 年齡牆點「帳號註冊 / 登入」。
2. 於彈窗點「帳號密碼登入」。
3. 輸入正確帳號、密碼，點「登入」。
4. 完成出現的滑塊驗證碼。
**預期**：登入彈窗關閉；出現會員元素（個人中心）；取得登入態 cookie。

## TC-02（P0）錯誤密碼登入失敗
**步驟**：同 TC-01，但密碼輸入錯誤值。
**預期**：維持在登入頁；顯示錯誤訊息（如「帳號或密碼錯誤」）；未取得登入態；無法進入會員區。

## TC-03（P1）空帳號 / 空密碼
**步驟**：帳號或密碼留空後點「登入」。
**預期**：阻擋送出或顯示必填提示；不送出登入請求。

## TC-04（P1）驗證碼未完成
**步驟**：輸入正確帳密送出後，**不**完成滑塊驗證碼。
**預期**：不應登入成功；停留在驗證碼 / 登入彈窗。

## TC-05（P2）帳號大小寫 / 前後空白
**步驟**：帳號加入前後空白或改變大小寫後登入。
**預期**：依後端規則處理（記錄實際行為，作為回歸基準）。

## TC-06（P2）登入態保留 / 重新整理
**步驟**：成功登入後重新整理頁面。
**預期**：仍維持登入狀態（cookie / session 有效）。

## TC-07（P2）安全性 — 注入字串
**步驟**：帳號/密碼輸入 SQL injection 樣式字串（如 `' OR '1'='1`）。
**預期**：登入失敗、無錯誤外洩、無異常行為。
